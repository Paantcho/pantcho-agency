Guia técnico completo para implementar animações e microinterações no Hubia. Cada regra tem demo ao vivo + código. Use como fonte única da verdade.

Instrução para o Cursor: Este documento define o comportamento de TODOS os elementos interativos do Hubia. Antes de implementar qualquer componente, consulte a seção correspondente aqui. Desvios precisam de aprovação explícita de Pantcho.
Fundação · 01

Tokens de Motion

Nunca use valores mágicos de duração ou curva. Sempre use os tokens CSS. Se não existe token para o caso, use o mais próximo e documente o desvio.

D
Duration Tokens
Velocidades disponíveis para transições
Token Valor Usar quando
--d1 100ms Micro-feedback: hover de cor, bordas. Ultra-rápido.
--d2 150ms Hover de botões, ícones, estados de foco.
--d3 200ms Dropdowns pequenos, tooltips, chips.
--d4 250ms Fade de elementos médios, overlay.
--d5 300ms Padrão principal. Modais, sidebar pill, tabs.
--d6 400ms Transições de página, elementos grandes.
--d7 500ms Animações elaboradas, ícones expressivos.
E
Easing Tokens
Material Design 3 — sem spring físico, sem bounce exagerado
Token Curva Usar quando
--ease-std cubic-bezier(0.4, 0, 0.2, 1) Movimento dentro da tela. Hover de botões e cards.
--ease-dec cubic-bezier(0.0, 0, 0.2, 1) Entrar na tela. Modais abrindo, dropdowns, toasts.
--ease-acc cubic-bezier(0.4, 0, 1.0, 1) Sair da tela. Modais fechando, toasts sumindo.
--ease-emp cubic-bezier(0.2, 0, 0.0, 1) Ações primárias. Pill da sidebar e das tabs.
--ease-exp cubic-bezier(0.34, 1.56, 0.64, 1) Ícones animados, badges, elementos lúdicos. Tem leve overshoot.

Botões

Todo botão tem 3 estados animados: normal → hover → pressed. A sensação deve ser de pressão física — sobe um pouco, depois afunda.

1

Passe o mouse e clique em cada botão

/* Todos os botões compartilham estas transições */ .btn { transition: background var(--d2) var(--ease-std), color var(--d2) var(--ease-std), transform var(--d2) var(--ease-exp); /* ease-exp = leve overshoot */ } .btn:hover { transform: scale(1.03); } /* sobe levemente */ .btn:active { transform: scale(0.96); } /* afunda no clique */

Componentes · 05

Modais

O modal nunca simplesmente aparece. Ele cresce de dentro para fora ao abrir, e encolhe e sobe ao fechar. O overlay de fundo também anima: escurece e foca gradualmente.

6
Modal Completo — Demo ao Vivo
Clique para abrir. Note o X ao passar o mouse.
DEMO
/* ── OVERLAY: escurece + blur ao abrir ── */ .modal-overlay { background: rgba(14,15,16, 0); backdrop-filter: blur(0px); transition: background var(--d4) var(--ease-dec), backdrop-filter var(--d4) var(--ease-dec); } .modal-overlay.open { background: rgba(14,15,16, 0.70); backdrop-filter: blur(12px); } /* ── MODAL CONTAINER: escala + sobe ao entrar ── */ .modal { transform: scale(0.88) translateY(20px); opacity: 0; transition: transform var(--d5) var(--ease-dec), opacity var(--d4) var(--ease-dec); } .modal.open { transform: scale(1) translateY(0); opacity: 1; } /* ── BOTÃO FECHAR: gira 90° ao hover ── */ .modal-close { transition: background var(--d2) var(--ease-std), transform var(--d2) var(--ease-exp); } .modal-close:hover { background: var(--i50); transform: rotate(90deg) scale(1.1); } .modal-close:active { transform: rotate(90deg) scale(0.9); } /* ── FECHAR: encolhe para baixo ao sair ── */ .modal.closing { transform: scale(0.92) translateY(10px); opacity: 0; transition: transform var(--d3) var(--ease-acc), opacity var(--d3) var(--ease-acc); /* acc = sai acelerando */ } /* JS — lógica de abrir/fechar */ async function closeModal() { modal.classList.add('closing'); // inicia animação saída overlay.classList.remove('open'); // fundo some junto await sleep(300); // espera animação terminar overlay.style.display = 'none'; }

Toasts & Notificações

Toasts entram de baixo com scale + blur, e saem deslizando para a direita. Nunca aparecem instantaneamente. Auto-dismiss após 3.5s.

8
Toasts — Demo ao Vivo
Dispare cada tipo
DEMO
/* Entrar: sobe + aparece + desblura */ @keyframes toast-in { from { opacity: 0; transform: translateY(20px) scale(0.88); filter: blur(4px); } to { opacity: 1; transform: translateY(0) scale(1); filter: blur(0); } } /* Sair: desliza para a direita */ @keyframes toast-out { to { opacity: 0; transform: translateX(30px) scale(0.92); } } .toast { animation: toast-in var(--d5) var(--ease-dec) forwards; } .toast.exiting { animation: toast-out var(--d4) var(--ease-acc) forwards; } /* JS — auto-dismiss com animação */ function dismiss(toast) { toast.classList.add('exiting'); // inicia saída setTimeout(() => toast.remove(), 400); // remove após anim } setTimeout(() => dismiss(toast), 3500); // auto-dismiss
Páginas · 01

Transições de Página

A troca de página nunca é abrupta. O conteúdo da página anterior sobe e desblura ao sair. O novo conteúdo desce e desblura ao entrar. A duração total deve ser de 300ms ou menos — rápido o suficiente para não atrasar, lento o suficiente para se perceber.

Regras · 01

Nunca Fazer

Esta lista de proibições é tão importante quanto as regras positivas. Cada item foi escolhido para manter o sistema elegante e consistente.

✕
Proibições Absolutas
Violações precisam de aprovação explícita de Pantcho
Nunca use animation-timing-function: ease ou linear para elementos de UI. Sempre use os tokens definidos: var(--ease-std), var(--ease-dec), etc.
Nunca use transition: all. Especifique exatamente o que deve animar: transition: transform 300ms, opacity 250ms. "All" anima propriedades que não devem animar e causa performance ruim.
Nunca use box-shadow para simular profundidade de card. O Hubia é flat. Separação visual é feita com cor de fundo e borda de 1px.
Nunca abra um modal sem overlay animado. O overlay de blur + escurecimento é obrigatório. Modais sem overlay parecem quebrados.
Nunca use visibility: hidden para esconder modais/toasts. Use opacity + transform para poder animar a entrada e saída.
Nunca use bounce ou spring em elementos de UI funcionais. O --ease-exp tem um leve overshoot controlado — isso é permitido apenas em ícones e badges. Navbars, modais e formulários usam --ease-emp ou --ease-dec.
Nunca deixe transições de página acima de 400ms no total. Exit: 200ms. Enter: 300ms. O usuário não pode sentir que está esperando.
Nunca anime width ou height diretamente. Para expandir/recolher, use max-height ou scale. Animações de width/height causam reflow e são lentas.
