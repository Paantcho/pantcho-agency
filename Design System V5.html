Ações · 02

Liquid Glass

Material translúcido inspirado na especificação Apple Liquid Glass (WWDC25). Refrata o conteúdo abaixo, tem specular highlights e responde ao ambiente visual. Usado apenas em elementos flutuantes.

🍎

Apple Liquid Glass — Referência oficial: developer.apple.com/documentation/technologyoverviews/liquid-glass

Princípios: material translúcido que refrata o conteúdo atrás (lensing), especular highlight no topo, opacidade adaptativa (mais opaco quando há conteúdo rolando por baixo).

✓ Usar glass em:
  • → Botões flutuantes (FAB, pill de ação)
  • → Navigation pill / dock flutuante
  • → Badges sobre fotos ou conteúdo visual
  • → Tooltips sobre áreas ricas em cor
  • → Overlays de controle de player
  • → Elementos que "flutuam" acima do layout
✗ Nunca usar glass em:
  • → Cards de dados ou KPI
  • → Tabelas e listas de conteúdo
  • → Formulários e campos de input
  • → Modais completos
  • → Sidebar ou navegação lateral
  • → Qualquer elemento sem fundo visual atrás
Tradução para Web
O efeito real de lensing da Apple usa Metal/CoreImage. Na web, traduzimos com backdrop-filter como melhor aproximação.
/* glass-light — sobre fundo claro/colorido */
backdrop-filter: blur(20px) saturate(1.8) brightness(1.1);
background: rgba(255,255,255,0.55);
border: 1px solid rgba(255,255,255,0.7);
/* specular highlight: */
inset 0 1px 0 rgba(255,255,255,0.9)

/* glass-dark — sobre fundo escuro/foto */
backdrop-filter: blur(20px) saturate(1.5) brightness(0.9);
background: rgba(14,15,16,0.28);
border: 1px solid rgba(255,255,255,0.14);

/* glass-limao — tint primário */
backdrop-filter: blur(20px) saturate(2.0) brightness(1.05);
background: rgba(215,255,0,0.22);
border: 1px solid rgba(215,255,0,0.35);
Glass Light — sobre fundos com textura
Glass Dark — sobre fundo escuro ou foto
Glass Limão — tint primário sobre foto/gradiente
Classe: btn-glass-limao (usar sobre fundos escuros com Limão como tint)
Navigation Pill — uso recomendado de glass

Misc · 02

Accordion

Chevron rotaciona 180°. max-height com ease-emp. Múltiplos podem estar abertos simultaneamente.

Todos os agentes leem WORKING.md, MEMORY.md e LESSONS.md antes de qualquer ação. Isso garante continuidade entre sessões e aprendizado incremental do sistema.
A janela de contexto nunca deve ultrapassar 40% de ocupação. Skills são carregadas sob demanda — nunca todas de uma vez. Subagentes são usados para trabalho pesado e retornam apenas o resumo.
Qualquer edição na aba Aparência cria uma nova versão automática (v1 → v2 → v3). Nenhuma versão é sobrescrita. Qualquer versão anterior pode ser restaurada criando uma nova versão com o conteúdo histórico.
Todo trabalho dos agentes segue 3 fases obrigatórias: PLAN — lista de tarefas antes de executar. EXECUTE — marcar conclusão, atualizar estado. REFLECT — validar qualidade enterprise antes de entregar. Sem REFLECT, sem entrega.
